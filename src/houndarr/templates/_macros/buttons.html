{#- Shared button macro for the four named low-variance call sites. Houndarr's button surface ranges from the dashboard's bespoke composite controls (split buttons, toggle pills, run-now triggers with embedded SVG and animations) to small affordances on the changelog modal, the admin maintenance / danger rows, and the shared confirm dialog. This macro covers ONLY the second group: low-variance call sites that share the daisyUI `btn`/`btn-soft`/`btn-primary` palette and one of three sizes. In scope (seven call sites across four templates): partials/changelog_modal.html "Don't show again", "Got it" partials/admin/confirm_dialog.html "Cancel", "Confirm" partials/admin/maintenance.html "Reset settings", "Clear logs" partials/admin/danger.html "Factory reset" Out of scope: the dashboard's split-button copy controls, the search-status mini-buttons, the shell-nav anchors (already centralised in _macros/htmx.html), and any element styled via the `.icon-btn` / `.btn-square` primitives; those are high-variance and routing them through a single macro would fork the class strings more than it saves. The macro composes three concerns through three independent seams: variant: maps to the daisyUI palette class string. Four values listed in the plan: primary, soft-neutral, soft-error, soft-success. Unknown variants fall back to primary so a typo lands on the safe-default surface. size: maps to the text + padding combo. Three values: sm = text-xs px-3 py-1.5 (admin maintenance + danger) md = text-sm px-3 py-1.5 (changelog modal) lg = text-sm px-3 py-2 (confirm dialog) Unknown sizes fall back to md. extra: trailing class string for layout / weight / one-off padding overrides. All seven call sites use it for at least one of: font-medium, font-semibold, shrink-0, or px-4 (the danger button widens the horizontal padding beyond the size default). `attrs` is an optional ordered mapping of html attribute name to value. Pass True for boolean attributes (autofocus, the bare `data-dismiss-confirm` marker on the cancel button) so the macro emits the attribute name without a value. Pass None to skip an attribute conditionally. Mapping iteration preserves insertion order on Python 3.7+ and Jinja2 mirrors that ordering, so callers control attribute order site-by-site. Macro-internal whitespace is pinned byte-equal by tests/test_templates/test_macros_buttons.py: the opening `` follows on its own line. The rendered class string emits the variant classes before the size combo and the optional `extra` trailing; Tailwind utilities are order-independent in the cascade so the rendered surface stays identical even when the consumer's own class ordering differs. -#} {#- btn: a daisyUI-style button. Args: variant: one of "primary" / "soft-neutral" / "soft-error" / "soft-success". Maps to the daisyUI palette class string. Defaults to "primary"; unknown variants fall back to "primary". size: one of "sm" / "md" / "lg". Maps to text + padding classes. Defaults to "md"; unknown sizes fall back to "md". extra: trailing class string appended after variant + size. Used for font-weight (font-medium, font-semibold), layout (shrink-0), and one-off padding overrides (px-4 on the danger button). Defaults to "" (none). attrs: optional dict mapping attribute name to value. Iterated in insertion order so callers control the rendered attribute order. Pass True for boolean attributes (autofocus, data-dismiss-confirm) and None to skip an attribute conditionally. -#} {%- macro btn(variant="primary", size="md", extra="", attrs=none) -%} {%- set variant_classes = { 'primary': 'btn btn-primary', 'soft-neutral': 'btn btn-soft btn-neutral', 'soft-error': 'btn btn-soft btn-error', 'soft-success': 'btn btn-soft btn-success', } -%} {%- set size_classes = { 'sm': 'text-xs px-3 py-1.5', 'md': 'text-sm px-3 py-1.5', 'lg': 'text-sm px-3 py-2', } -%} {%- set _variant = variant_classes.get(variant, variant_classes['primary']) -%} {%- set _size = size_classes.get(size, size_classes['md']) -%} {{ caller() }} {%- endmacro %}
