{#- Shared HTMX shell-fetch macros. Houndarr's single-page shell swaps route bodies into `#app-content` via HTMX-driven anchor tags. Every such anchor carries the same four wire attributes (`hx-get`, `hx-target`, `hx-swap`, `hx-push-url`), plus an `href` fallback so native full-page navigation still works when HTMX is absent (first paint, JS disabled, copy-link). Track F.1 centralises the wire shape here; F.2 migrates the two consumers (`partials/shell_nav_links.html`, base.html's header logo) without changing the rendered markup. `shell_nav_link` extends the base with the `data-shell-*` hooks app.js uses for active-state management, and encodes the conditional active/inactive class so every call site renders byte-identical markup. `is_active` is a bool computed by the caller, since the /settings pill lights up for both `/settings` and any `/settings/*` sub-path; a path-equality test in the macro would not capture that rule. `hx_shell_fetch` accepts arbitrary child content via `{% call %}` and is used by the header logo, where the anchor wraps an img and a text label with a bespoke class string. -#} {#- shell_nav_link renders one of two visual variants. variant='pill': desktop header pill-container tab (default). variant='mobile': drawer row used inside #mobile-nav-menu.
