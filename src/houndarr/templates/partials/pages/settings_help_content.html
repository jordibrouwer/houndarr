Instance Settings Help

Quick reference for each Add/Edit Instance control, with defaults and safe-operating guidance.

Back to Settings

Recommended mindset

Start conservative first. Increase one control at a time and watch logs for a full day before increasing again.

Missing Search Controls

Primary workload. Increase slowly only if indexers remain healthy.

Batch Size

Default: 2

Maximum missing items considered per cycle.

Sleep (minutes)

Default: 30

Wait time between cycles for each instance.

Hourly Cap

Default: 4

Maximum successful missing searches per hour (0 = unlimited).

Cooldown (days)

Default: 14

Minimum delay before retrying the same missing item.

Post-Release Grace (hrs)

Default: 6

Hours to wait after an item's release date before searching. Items still within this window are skipped. Unreleased items (no release date yet, or release date in the future) are always skipped regardless of this setting.

Missing Search Mode

Per-app setting

Sonarr, Lidarr, Readarr, and Whisparr v2 each offer an advanced search mode alongside their default item-level search. The advanced mode groups items by a parent context (season, artist, author) to issue fewer, broader commands. Radarr and Whisparr v3 always search at the movie level.

Advanced warning: broader searches may produce noisier results. Sonarr/Whisparr v2 season search is not pack-only and may still pull singles.

Queue Backpressure

Prevents Houndarr from adding work when the download client is already busy.

Queue Limit

Default: 0 (disabled)

When set above zero, Houndarr checks the instance's download queue before each cycle. If the total queue count meets or exceeds this limit, the entire cycle is skipped. If the queue endpoint is unreachable, the search proceeds normally.

Cutoff Upgrade Controls

Lower priority than missing. Keep conservative unless backlog is stable.

Cutoff search

Default: Off

Enables cutoff-upgrade pass for items below quality cutoff.

Cutoff Batch

Default: 1

Maximum cutoff items considered per cycle.

Cutoff Cooldown

Default: 21

Minimum delay before retrying the same cutoff item.

Cutoff Cap

Default: 1

Maximum successful cutoff searches per hour (0 = unlimited).

Cutoff has its own budget and cooldown so it does not compete with missing searches.

Library Upgrade Controls

Lowest priority. Re-searches items that already meet cutoff for better releases. Keep off unless missing and cutoff backlogs are stable.

Upgrade search

Default: Off

Enables upgrade pass for library items that already have files and meet your quality cutoff.

Upgrade Batch

Default: 1

Maximum upgrade items considered per cycle. Hard cap: 5.

Upgrade Cooldown (days)

Default: 90

Minimum delay before retrying the same upgrade item. Minimum: 7 days.

Upgrade Cap

Default: 1

Maximum successful upgrade searches per hour (0 = unlimited). Hard cap: 5.

Upgrade Search Mode

Per-app setting

Sonarr, Lidarr, Readarr, and Whisparr v2 each offer an advanced search mode for the upgrade pass, independent of the missing search mode. Radarr and Whisparr v3 always use movie-level search.

Upgrade has its own budget, cooldown, and offset rotation so it does not compete with missing or cutoff searches. All three passes (missing, cutoff, upgrade) rotate through items across cycles.

Safe Starting Preset

Batch Size: 2
Sleep: 30 min
Hourly Cap: 4
Cooldown: 14 days
Post-Release Grace: 6 hrs
Queue Limit: 0 (disabled)
Cutoff Search: Off
Cutoff Batch: 1
Cutoff Cooldown: 21 days
Cutoff Cap: 1
Upgrade Search: Off
Upgrade Batch: 1
Upgrade Cooldown: 90 days
Upgrade Cap: 1

Need More Throughput?

  1. Increase Batch Size slightly.
  2. Lower Sleep slightly.
  3. Increase Hourly Cap only if indexers remain healthy.
  4. Enable Cutoff search after missing backlog is under control.
  5. Enable Upgrade search last, only after both missing and cutoff are stable.

Change one control at a time and observe logs for at least 24 hours.
